# 短剧分镜规范

> 对齐 [AI 短剧制作平台 v2 设计文档](../../../AI短剧制作平台_全功能设计文档_v2.md)的 Sequence ID 架构。

## 1. Sequence ID

格式：`EP{2位集数}_SC{2位场次}_SH{3位镜头号}`

| 段 | 含义 | 范围 |
| --- | --- | --- |
| EP | 集数 | 01-99 |
| SC | 场次 | 01-99，单集内唯一 |
| SH | 镜头号 | 001-999，单场内严格递增 |

示例：`EP01_SC03_SH012`。

本工具默认 `EP01_SC01`，由 `--episode` 和 `--scene-no` CLI 参数覆盖。

## 2. 镜头要素

每个 SH 必须含：

| 字段 | JSON 键 | 取值 |
| --- | --- | --- |
| 景别 | `shot_type` | 远景 / 中景 / 近景 / 特写 / 大特写 / 过肩 OTS / POV |
| 机位运动 | `camera_hint` | 固定 / 推 / 拉 / 摇 / 移 / 跟 / 升降 / 手持 |
| 节拍 | `beat` | 施压 / 退让 / 转折 / 揭示 / 沉默 / 爆发 / 落定 |
| 时长估计 | `duration_est` | 秒，整数 |
| 动作 | `action` | 一句可拍到的画面描述 |
| 对白 | `dialogue` | `[{speaker, text}, ...]`，可为空 |
| 人物 | `characters` | 出现在镜头中的角色 id 列表 |

## 3. 节奏原则

1. **景别交错**：连续 ≥ 2 个相同景别会让画面发木。单段长对白拆为 2-3 个 SH。
2. **节拍换挡**：每 3-5 个 SH 出现一次反转/揭示节拍，避免冲突单向滑落。
3. **首末锚定**：开场 SH 必须建立空间（建议远景或全景），末镜必须落定情绪（建议特写或大特写）。
4. **动作锚点**：尽量让每个 SH 至少有一个可拍到的物理动作，纯静态对白只在节拍 `沉默` 时出现。
5. **时长**：对白镜头按"原句汉字数 × 0.25 秒"估算下限，动作镜头 2-4 秒。

## 4. Markdown 样例

```
# EP01_SC01 — 咖啡馆对峙
**场景头**：内 · 加州咖啡馆 · 黄昏  
**人物**：乔布斯、马斯克

## SH001  中景过肩 · 推
（动作）乔布斯放下杯子，目光锁定马斯克
乔布斯：「你以为火箭就是未来？」
节拍：施压 · 4s

## SH002  特写 · 固定
马斯克：「不是以为，是必然。」
镜头建议：从马斯克瞳孔反光中拉远
节拍：转折 · 3s

## SH003  远景 · 升降
（动作）窗外橘色落日缓缓沉入地平线，两人静默
节拍：沉默 · 5s
```

## 5. JSON 样例

```json
[
  {
    "sequence_id": "EP01_SC01_SH001",
    "shot_type": "中景过肩",
    "characters": ["jobs", "musk"],
    "action": "乔布斯放下杯子，目光锁定马斯克",
    "dialogue": [{ "speaker": "jobs", "text": "你以为火箭就是未来？" }],
    "beat": "施压",
    "camera_hint": "推",
    "duration_est": 4
  },
  {
    "sequence_id": "EP01_SC01_SH002",
    "shot_type": "特写",
    "characters": ["musk"],
    "action": null,
    "dialogue": [{ "speaker": "musk", "text": "不是以为，是必然。" }],
    "beat": "转折",
    "camera_hint": "固定",
    "duration_est": 3
  }
]
```

## 6. 与 AI 短剧制作平台对接

`screenplays/<sessionId>.json` 的数组可直接喂给"剧情编排器 Agent"作为初始 Sequence 列表。后续资产设计、视频生成、音画对齐都通过同一 Sequence ID 串联（"全局数据粘合剂"）。
